# GNSS Urban PNT MiniLab - Brief Update for Prof. Bing Xu

Dear Prof. Xu,

Thank you again for your earlier advice on preparing for satellite positioning and navigation research. Following your suggestion, I started from the fundamentals of GNSS pseudorange positioning and then built a small Python project to connect the textbook equations with simplified urban PNT problems such as multipath, NLOS, robust estimation, and LEO-like aiding. The project is called **GNSS Urban PNT MiniLab**. It is not intended to be a research-grade GNSS receiver, but rather a transparent learning and experimentation platform that shows the preparation I have started before joining PolyU.

The first part of the project implements a simplified GNSS pseudorange positioning solver. The measurement model is formulated as pseudorange equals geometric range plus receiver clock bias plus measurement error. In the current 2D toy setting, the solver estimates receiver position and a receiver clock-bias term expressed in meters. I implemented iterative ordinary least squares as the baseline, and then added weighted least squares, Huber IRLS, Cauchy IRLS, and a simple residual-exclusion baseline. This helped me understand how pseudorange residuals, weighting, and robust loss functions influence the final solution when some measurements become biased.

The second part focuses on synthetic urban degradation. I added Gaussian pseudorange noise, multipath / NLOS-like positive range biases, and spoofing-like drift applied only to synthetic pseudorange arrays. I also added a simple 2D urban-canyon model where rectangular building blocks can mark a receiver-to-source line as LOS or NLOS. This is still a very simplified geometry model, but it allowed me to generate controlled scenarios and compare how OLS, WLS, robust LS, and residual exclusion behave under different levels of measurement degradation. One lesson from these experiments is that robust estimation is not automatically better unless the weighting model, threshold, and measurement geometry are suitable.

The third part studies geometry through DOP / HDOP. I implemented a simplified 2D geometry matrix for the state `[x, y, clock_bias_m]` and used it to compare GNSS-only and GNSS plus additional moving ranging sources. These moving sources are described as LEO-like only in the conceptual sense: they are abstract 2D ranging beacons, not real LEO orbit predictions or signal models. The clean-geometry experiment shows that additional ranging sources can reduce HDOP, while the degraded-measurement experiment shows that better geometry alone is not sufficient if biased pseudorange measurements are not properly handled. This is a useful lesson for thinking about integrated PNT rather than treating LEO aiding as a simple guaranteed improvement.

The fourth part is a small pseudorange-correlogram-inspired toy experiment. I read about the idea of evaluating candidate receiver states using pseudorange consistency and signal-quality-related weighting, and implemented a simplified candidate-state scoring experiment in Python. The searched state is `[x, y, clock_bias_m]`, and the weights are synthetic C/N0-like values generated inside the project. This is only a conceptual replication and does not use real UrbanNav RINEX observations, IF samples, real C/N0, real ephemerides, or the full direct-position-estimation pipeline. Nevertheless, it helped me understand why a candidate-search approach may behave differently from direct least-squares fitting under NLOS-dominated measurements.

For reproducibility, I organized the project into core modules, experiment scripts, documentation, tests, and selected result figures. The main scripts can regenerate the synthetic metrics and plots, and the unit tests check the solver behavior, DOP calculation, robust estimation, synthetic urban environment, GUI result loading, and the pseudorange-correlogram toy module. I also prepared a small GitHub-facing `results/selected/` folder with five representative figures: robust solver comparison, synthetic urban LOS/NLOS demonstration, degraded LEO-like aiding comparison, pseudorange-correlogram score map, and an error time-series comparison.

The current limitations are clear. The project uses synthetic data, a 2D toy geometry, simplified satellite and LEO-like source positions, and educational robust-estimation baselines. It does not process real RINEX data, IF samples, real ephemerides, real LEO orbit models, carrier phase, atmospheric corrections, or real receiver logs. It also does not include RF generation, SDR transmission, or any real spoofing capability. My purpose at this stage is to build a solid foundation and to make my learning process visible and reproducible.

My next steps are to move from this toy environment toward more realistic offline data processing: using real public GNSS datasets, implementing a more standard coordinate frame and ephemeris-based satellite geometry, comparing robust LS with more rigorous GNSS/INS or factor-graph approaches, and studying LEO PNT literature more systematically. I would be very grateful for any advice on whether this direction is suitable for an MSc dissertation topic, and which parts I should strengthen before starting a more serious research project.

Best regards,

Yalin Han
